# CarRent - Application Web de Location de Voitures

## &#x20;Présentation

CarRent est une application web permettant aux utilisateurs de rechercher, réserver et gérer la location de véhicules en ligne de manière simple et rapide.

L'application repose sur une architecture moderne séparant le frontend et le backend :

* un frontend en React pour une interface utilisateur dynamique et fluide
* un backend en Java pour gérer la logique métier, les données et la sécurité

Cette séparation permet d'obtenir une application scalable, maintenable et performante.

## Objectifs du projet

* Simplifier le processus de location de voiture en ligne
* Offrir une interface moderne et intuitive
* Garantir une gestion fiable des réservations
* Mettre en place une architecture web professionnelle

## Fonctionnalités utilisateur

Les utilisateurs peuvent :

* &#x20;Rechercher un véhicule selon différents critères  (date, type de véhicule, disponibilité)
* Consulter les informations des voitures
  * modèle
  * prix par jour
  * caractéristiques
  * disponibilité
* &#x20;Réserver un véhicule directement en ligne
* &#x20;Créer et gérer leur compte utilisateur
* &#x20;Consulter l'historique de leurs locations

## Fonctionnalités administrateur

L'administrateur dispose d'un espace de gestion permettant de :

* &#x20;Ajouter de nouveaux véhicules
* &#x20;Modifier les informations des voitures
* &#x20;Supprimer un véhicule
* &#x20;Consulter et gérer les réservations
* &#x20;Gérer les utilisateurs





## Améliorations si possible

* Intégration d'un système de paiement en ligne
* Ajout d'une application mobile
* Système d'avis et de notation des véhicules
* Géolocalisation des voitures disponibles

## Conclusion

CarRent propose une solution moderne de location de voitures, combinant une interface utilisateur réactive grâce à React et une gestion robuste des données via un backend Java.

Cette architecture garantit une application performante, sécurisée et évolutive.

## Architecture technique

L'application est construite selon une architecture client / serveur.

### Frontend

* React
* Gestion des états et des composants
* Communication avec l'API backend

### Backend

* Java
* API REST
* Gestion de la logique métier
* Gestion des utilisateurs et des réservations

### Base de données

* Stockage des informations :
  * utilisateurs
  * véhicules
  * réservations

## Sécurité

* Authentification des utilisateurs
* Gestion des rôles (utilisateur / administrateur)
* Validation des données côté serveur
* Protection des routes API
